# 音小鲸关爱自检 V7：专家质询后的结果承接与品牌体验页设计

## 模拟专家质询

本节为基于专业角色的设计审查，不代表真实医疗机构或具名专家背书。

### 听力专业审查

- “改善听力”容易被理解为医疗效果承诺，页面统一使用“帮助日常交流听得更轻松”“提供日常聆听支持”。
- 自检只反映生活场景中的困扰频率，不能表示听力损失程度。
- 产品体验必须位于专业检查边界之后；突发单侧变化、耳痛、眩晕或耳鸣明显加重继续提示及时就医。

### 用户心理审查

- 结果页三个同权按钮会形成选择困难，也容易让“沟通、体验、再测”被理解为同一件事。
- 用户刚看到风险结果时需要缓冲语言，不能立即进入产品推介。
- 先给沟通方法，再给体验选择，最后给再次确认的路径，更符合子女帮助父母的心理顺序。

### 交互逻辑审查

- 专家解读结束后先出现过渡句，再分三张独立承接卡。
- 卡片顺序固定为沟通、体验、再测；每张卡只有一个明确动作。
- 陪测引导默认折叠，点击再测卡后原位展开，不与其他按钮挤在一起。

### 文案审查

- 避免“问题严重”“改善听力”“最佳方案”等诊断化或绝对化表达。
- 使用“想帮助父母听得更轻松”“不知道怎么开口”“先亲自体验，再判断是否适合”等低压力语言。
- 产品特点只陈述功能方向和体验价值，不保证个人效果。

### UI 审查

- 首页内容集中在过窄文字柱会造成两侧空旷，应扩大标题与按钮占宽。
- 结果行动需通过卡片背景、间距和标题层级形成明确区隔。
- 产品页应先完成品牌确认，再讲产品定义、主图、特点与试听。
- 面向长辈阅读，特点标题和正文需要增大，图标也应同步放大。

## 首页：中轴聚合

- 所有内容沿屏幕中轴对齐。
- 顶部不再占据独立导航高度；Logo 直接进入主内容组。
- “听见关爱”四字一行，位于视觉中心，字号显著放大，字间距适当展开，宽度约占屏幕 70%。
- 痛点、说明、按钮和信任文字紧随标题排列，内容组整体居中但不依赖大面积空白。
- 主按钮宽度加大，避免页面形成窄文字柱。

## 结果页：三张独立承接卡

专家解读之后显示过渡语：

“看到这些变化，不必急着下结论。下一步，可以先从一次不带压力的沟通开始。”

### 沟通卡（主行动）

- 标题：“想帮助父母听得更轻松，却不知道怎么开口？”
- 按钮：“看看怎么和父母说”
- 跳转沟通页面。

### 体验卡（次行动）

- 标题：“如果父母也想听清一点，可以先亲自体验，再判断是否适合。”
- 按钮：“去体验这个方案”
- 跳转产品体验页面。

### 再测卡（辅助行动）

- 标题：“如果你拿不准，可以和父母一起再测一次。”
- 按钮：“今天和父母一起测”
- 原位展开四步陪测引导。

三张卡不得使用连续同色大按钮；主卡深绿色，体验卡暖白描边，再测卡采用低对比浅灰绿色。

## 沟通页面

- 保留七种父母反馈应对、顶部返回结果和底部体验入口。
- 体验入口文案统一为“去体验这个方案”。
- 不包含复制沟通话术功能。

## 产品体验页

页面顺序固定为：

1. Logo 与品牌名“音小鲸 AuraClear”；
2. 产品定义“开放式骨传导辅听耳机”；
3. 产品标题与一句介绍；
4. 清晰立体主图；
5. 五个特点；
6. 适用边界；
7. 免费试听电话与安心承诺。

品牌区域使用圆形或圆角 Logo，不依赖外部资源。五项特点标题不小于 19px，正文不小于 15px，图标不小于 50px。产品页继续使用暖白背景，主图保持清晰、立体但不使用混合模式或主体模糊。

## 状态、返回与隐私

- 检测答案、结果、卡片展开状态仅保存在当前页面。
- 结果可返回第 10 题；沟通可返回结果；产品可返回沟通。
- 页面无表单、无二维码、无资料上传、无支付行为。

## 验收标准

- 首页 Logo、标题、痛点、说明和按钮沿中轴排列，标题宽度明显增加。
- 结果专家解读后出现过渡语和三张独立卡片，顺序为沟通、体验、再测。
- 三张卡的文案和按钮完全符合本设计，陪测引导只由再测卡展开。
- 产品页顶部出现 Logo、音小鲸 AuraClear 和产品定义。
- 五项特点标题、正文和图标整体放大。
- 所有医疗边界、返回路径与隐私边界保留。
- GitHub Pages 公网返回 200 并显示 V7 内容。
